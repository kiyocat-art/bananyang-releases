## 1. 🥶 냉정한 자체 감사 (내부 현실 점검)

### 성능 및 안정성

* **현재 상태**: 최근의 메모리 최적화(Blob 정리, 히스토리 제한)로 앱 충돌은 안정화되었습니다. 하지만 근본적인 아키텍처(React/DOM 기반 무한 캔버스)는 WebGL/GPU 네이티브 경쟁사들보다 **태생적으로 무겁습니다**.
* **불편한 진실**:
  * **렌더링**: DOM 기반 캔버스에 고화질 이미지 100개 이상을 띄우고 드래그하면, Krea나 Leonardo의 엔진에 비해 버벅거림(Stuttering)이 발생할 수밖에 없습니다.
  * **VRAM**: Electron의 공유 GPU 프로세스에 의존합니다. 무거운 로컬 생성 작업과 렌더링이 자원을 두고 경쟁하여 "흰색 화면(WSOD)"이 발생하며, 이는 엔진 효율이 아닌 공격적인 리소스 정리로 간신히 막고 있는 실정입니다.
  * **시동 속도**: 초기 로딩 시간은 "허용 가능한 수준"이지만, Krea처럼 "즉각적(Instant)"이지 않습니다.

### 사용성 및 UX

* **복잡성**: 우측 패널이 너무 빽빽합니다. "의상 창의성(Costume Creativity)", "합성 모드", "레이어 가중치" 등은 강력하지만 신규 사용자에게는 압도적입니다.
* **워크플로우**: "무한 캔버스"는 강력한 기능이지만, **Miro**나 **InvokeAI의 Unified Canvas** 같은 최신 툴들이 가진 "자석(Magnetic)" 기능이나 "자동 레이아웃" 같은 스마트함이 부족합니다.
* **접근성**: 단축키는 존재하지만 찾기 어렵습니다. 처음 사용자를 위한 "가이드 투어"가 없습니다. 마치 엔지니어가 엔지니어를 위해 만든 도구처럼 느껴집니다.

## 2. 🌍 시장 지형도 (경쟁사 분석)

2025-2026년 시장을 스캔했습니다. 시장은 크게 세 가지 지배적인 카테고리로 나뉘었습니다:

### A. "웹 캔버스" 거인들 (직접 경쟁자)

* **Leonardo.ai**: 가장 큰 위협입니다. 그들의 "Canvas Editor"는 클라우드에서 실행되어(사용자 VRAM 소모 0), 인페인팅/아웃페인팅을 완벽하게 지원하며, 거대한 커뮤니티 스타일 라이브러리를 보유하고 있습니다.
  * *판정*: 접근 용이성과 속도 면에서 우리를 압도합니다.
* **InvokeAI (Unified Canvas)**: "프로" 옵션입니다. 2025년 업데이트로 **객체 선택(SAM2)**과 **조정 레이어(Adjustment Layers)**를 추가했습니다. 사실상 브라우저 안에 포토샵을 재건했습니다.
  * *판정*: 기능의 깊이와 편집 정밀도에서 우리를 압도합니다.

### B. "실시간" 스피드스터

* **Krea.ai**: "실시간" 시장을 장악했습니다. 사용자가 졸라맨을 그리면 0.5초 만에 4K 예술 작품이 됩니다.
  * *판정*: 바나냥이 백엔드를 완전히 뜯어고치지 않는 한(LCM/Turbo 모델 + 웹소켓 도입), 이 부분에서 경쟁할 수 없습니다.

### C. "데스크톱 강자들"

* **ComfyUI**: 백엔드의 제왕입니다. 노드 기반입니다. 사용자가 "제어(Control)"를 원한다면 Comfy를 씁니다.
* **Fooocus**: "쉬운 데스크톱"의 제왕입니다. 무료이고 빠르며 사실상 매우 간단합니다(프롬프트 창 하나).
  * *판정*: 우리는 어중간한 위치에 끼어 있습니다. Fooocus보다는 복잡하고, ComfyUI보다는 유연성이 떨어집니다.

## 3. ⚠️ 갭 분석 (우리가 위험한 이유)

| 기능 | 경쟁사 (표준) | 바나냥 AI (현재) | 상태 |
| :--- | :--- | :--- | :--- |
| **캔버스 엔진** | WebGL / GPU 가속 | HTML DOM / CSS Transform | 🔴 대규모 시 랙 발생 |
| **생성형 채우기** | 객체 인식(SAM2), 문맥 인식 | 마스크 기반 인페인팅 | 🟠 구식 |
| **실시간성** | < 1초 지연 (Krea, LCM) | 큐 기반 (~5-10초) | 🔴 느림 |
| **접근성** | 웹 (어디서나 접속) | 데스크톱 (설치 필요) | 🟠 진입장벽 높음 |
| **가격 정책** | 부분 유료 / SaaS | 로컬 하드웨어 비용 | 🟡 니치한 매력 |

### 제품 포지셔닝의 "불쾌한 골짜기 (Uncanny Valley)"

바나냥은 현재 **생산성의 "불쾌한 골짜기"**에 빠져 있습니다:

* 캐주얼 사용자에게는 충분히 단순하지 않음 (Midjourney/Fooocus 사용).
* 기술 전문가에게는 충분히 강력하지 않음 (ComfyUI 사용).
* 프로에게는 충분히 세련되지 않음 (Photoshop/InvokeAI 사용).

## 4. 💼 사업 타당성 및 전략

### 현 상태로 생존 가능한가? -> **절대 불가능 (Hard NO).**

Leonardo나 Midjourney 같은 "일반 이미지 생성기"와 정면 승부하는 것은 질 수밖에 없는 싸움입니다. 그들은 수백만 달러의 컴퓨팅 자금을 가지고 있습니다. 우리는 로컬 Electron 앱일 뿐입니다.

### 피벗 전략 (생존과 성장)

살아남기 위해, 바나냥은 "모든 것"이 되려는 시도를 멈추고 "특정 분야의 최고 도구"가 되어야 합니다.

#### 옵션 A: "연속성" 전문가 (웹툰/게임 에셋)

* **이유**: 현재의 생성기들은 *일관성(Consistency)* 유지에 어려움을 겪습니다.
* **조치**:
  * **캐릭터 시트**와 **의상 고정(Costume Persistence)**에 100% 집중합니다.
  * 경쟁사에 없는 기능을 추가합니다: "원클릭 턴어라운드 뷰", "의상 관리자", "에셋 내보내기(JSON/Spine 지원)".
  * 무한 캔버스를 단순 이미지 보관소가 아닌 "스토리보드" 도구로 만듭니다.

#### 옵션 B: "로컬 프라이버시" 방패

* **이유**: 기업 사용자들은 IP 유출 우려로 클라우드 도구(Leonardo/Midjourney)를 싫어합니다.
* **조치**:
  * "로컬 AI계의 포토샵"으로 마케팅합니다.
  * 보안, 오프라인 기능, "데이터 학습 없음" 보장에 올인합니다.
  * "소셜/채팅" 기능을 제거하고 "에셋 관리" 도구를 다듬습니다.

### 📝 즉시 실행 권장 사항

1. **UX 개편**: 우측 패널 옵션의 80%를 "고급" 토글 아래로 숨깁니다. 기본값은 "스케치 -> 이미지"로 설정합니다.
2. **성능**: 엔진을 재작성할 수 없다면, **가상화(Virtualization)**(화면 밖 이미지 언마운트)를 즉시 구현해야 합니다. 현재의 "임시 해결책"은 확장성을 해치고 있습니다.
3. **차별화**: 지금 당장 독보적인 기능을 통합해야 합니다. (예: 게임 개발자를 위한 "자동 배경 제거 & 스프라이트 시트 패커").
